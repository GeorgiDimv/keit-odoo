# KEIT Ltd Bulgaria — Odoo 18 ERP

Self-hosted Odoo 18 Community + OCA modules with Bulgarian accounting
localization. Covers **Accounting, Inventory, Sales, Purchase, and
Manufacturing**. Single-tenant, runs on your own VM behind your firewall.

## What's included

- **Odoo 18 Community** (no per-user license fees)
- **Bulgarian localization** (`l10n_bg`) — chart of accounts, VAT, legal
  document sequences
- **16 OCA module repositories** — financial reports, bank reconciliation,
  CAMT statement import, SEPA payments, asset management, stock valuation,
  and more
- **Manufacturing** — BoMs, manufacturing orders, make-to-order routes,
  lot/serial traceability
- English user interface (Bulgarian accounting compliance retained)

## Stack layout

```
docker-compose.yml    Odoo 18 + PostgreSQL 16
config/odoo.conf      Odoo server configuration
setup-oca.sh          Clones the 16 OCA module repos into addons/
addons/               OCA modules (regenerated by setup-oca.sh)
Makefile              All operations (make help)
guides/               Role guides for the team
  ADMIN_GUIDE.md
  INVENTORY_GUIDE.md
  PRODUCTION_GUIDE.md
  ACCOUNTING_GUIDE.md
```

## Requirements

- A Linux VM (or Docker host) with ≥ 4 GB RAM, ~10 GB free disk
- Docker + Docker Compose (or a bare-metal install — see deployment notes)
- Ports 8069 (web) and 8072 (long-polling) available

## Bring the stack up

```bash
cp .env.example .env     # set real secrets before production
make oca                 # clone OCA modules (~10 min, ~400 MB)
make up                  # start Odoo + PostgreSQL
make logs                # wait for "HTTP service running on 0.0.0.0:8069"
```

Then open the web UI and create / restore the database.

## Common operations

```bash
make up          # start
make down        # stop
make restart     # restart Odoo only
make logs        # tail logs
make psql DB=...        # SQL console
make update MODULE=x DB=...   # install / upgrade a module
make stats       # RAM / CPU
make help        # list all targets
```

## Production hardening (before go-live)

- Replace `.env` defaults with strong secrets
- In `config/odoo.conf`: set `list_db = False`, add `dbfilter`, set a
  strong `admin_passwd`, set `proxy_mode = True`
- Put a reverse proxy (Caddy / nginx) in front with HTTPS
- PostgreSQL is not exposed to the network (no published port) — keep it that way
- Daily backups: database dump + filestore, copied off the VM, restore-tested

## Important

- **Never uninstall `l10n_bg` or `l10n_bg_ledger`** — the Bulgarian chart
  of accounts and VAT compliance depend on them.
- Install additional modules one at a time and watch the logs.
- Do not modify Odoo core source; all customization lives in addon modules.

## Out of scope (separate engagements)

- Payroll
- Direct submission to the National Revenue Agency (NRA)
- Live banking API integrations (file-based CAMT import is included)
